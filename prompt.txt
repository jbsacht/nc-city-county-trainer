I would like to make a full stack web application to learn all of the cities in North Carolina (the 551 or so that are on wikipedia). I think the flow should be this: first, select if you want to go in alphabetical order, random order, descending population order, or date founded order (either direction for all). Then going through all cities, you are prompted with the city name, and are shown a blank map of NC with county outlines. You must first click on a county, which then will show you a blown up map of the county (togglable highways on the map would be great if possible). You must then click on where in the county the city is, and if you are close enough, say 5 miles (make this in a menu settings configurable), you are awarded as correct. Also a setting by defult yes should be that county seats will have a star next to their name so you know it is one. Once you have answered, it should show a point for the exact location of the city, with all of the info like population date founded etc, and then a next button to continue. Show a score at the top, but the main priority is to learn. 
Have a secondary mode that instead of the city name, you are just given the county name and must select where it is. Then have a reverse of this as well as an option where a county is highlighted and you must name the county.
